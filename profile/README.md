## 📋 Overview

<a href="#">FontExplorer X</a> provides professional font library management for designers, typographers, and studios who work with large collections of typefaces — organizing, activating, and accessing fonts in ways that macOS's built-in font management cannot handle at professional scale.

The <a href="#">font library organization</a> structures collections into named sets, folders, and smart sets — grouping fonts by project, client, style, foundry, or any other criteria — making large libraries navigable without hunting through hundreds of activated fonts cluttering the system font menu. <a href="#">Selective activation</a> keeps only project-relevant fonts active in the system, reducing font menu clutter and eliminating conflicts between similar-named fonts from different foundries that cause application behavior problems when large numbers of fonts are simultaneously active.

<p align="center">
  <img src="https://static.macupdate.com/screenshots/330048/m/fontexplorer-x-pro-screenshot.png?v=1646234771" alt="FontExplorer X screenshot"/>
</p>

<a href="#">Auto-activation</a> detects which fonts a document requires and activates them automatically when the document opens, eliminating the manual font activation step for historical and client project documents that use fonts not kept permanently active. <a href="#">Conflict detection</a> identifies font name conflicts — multiple fonts with the same PostScript name from different foundries — and provides resolution tools to prevent the application behavior issues that conflicts cause.

The <a href="#">font preview tools</a> display custom sample text in any installed font at configurable sizes for comparison and selection during design work. <a href="#">Type specimen generation</a> creates formatted specimen documents showing a font family's full range of weights, styles, and character sets for presentation, reference, and client communication.

## 🤔 FAQ

<details>
<summary>How does auto-activation work?</summary>

FontExplorer X uses plugins for Adobe Creative Cloud applications that detect which fonts a document requires when it opens. If those fonts are in the managed library but not active, FontExplorer X activates them automatically before the document fully loads.

</details>

<details>
<summary>What is a font name conflict and why does it matter?</summary>

A font name conflict occurs when two different font files share the same PostScript name. Applications and the system can only load one of them, leading to unpredictable font rendering and inconsistency between design files and output.

</details>

<details>
<summary>Can FontExplorer X manage fonts that are not installed?</summary>

Yes. Fonts in the FontExplorer X library are managed independently of system-installed fonts. Only fonts you explicitly activate are available system-wide, keeping the system font menu clean.

</details>

<details>
<summary>Does FontExplorer X support variable fonts?</summary>

Yes. Variable fonts are supported in the library and activation system, with font information displaying the variable axes and ranges the font supports.

</details>
